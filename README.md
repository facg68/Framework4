# 🐺 Framework4 (F4)

## ENGLISH

### What is F4?

Framework4 (F4) is a lightweight hybrid web framework built on classic ASP that rethinks the relationship between server and client.

It is not about rendering pages.  
It is about controlling *when* and *how* views change.

> Server defines the views.  
> Client decides when to switch them.

---

### Philosophy

Most traditional approaches fall into two categories:

- Server-driven (full reloads)
- Client-heavy (SPAs with complex state)

F4 introduces a hybrid model:

- The server prepares structured views
- The client controls transitions and rendering timing
- No unnecessary complexity
- No heavy frameworks required

---

### Key Concepts

- **View-based architecture**
- **Client-controlled rendering flow**
- **Minimal dependencies**
- **Classic ASP as a flexible core (no imposed limits)**

---

### Why F4?

Because sometimes:

- You don’t need a SPA
- You don’t want full page reloads
- You want control without overengineering

F4 gives you that middle ground.
