## ESPAÑOL

### ¿Qué es F4?

Framework4 (F4) es un framework web híbrido y liviano basado en ASP clásico que replantea la relación entre servidor y cliente.

No se trata de renderizar páginas.  
Se trata de controlar *cuándo* y *cómo* cambian las vistas.

> El servidor define las vistas.  
> El cliente decide cuándo cambiarlas.

---

### Filosofía

La mayoría de enfoques tradicionales caen en dos extremos:

- Control total del servidor (recargas completas)
- Control total del cliente (SPAs complejas)

F4 propone un modelo híbrido:

- El servidor prepara las vistas
- El cliente controla el flujo de renderizado
- Sin complejidad innecesaria
- Sin frameworks pesados

---

### Conceptos Clave

- **Arquitectura basada en vistas**
- **Control del render desde el cliente**
- **Dependencias mínimas**
- **ASP clásico como núcleo flexible (sin restricciones)**

---

### ¿Por qué F4?

Porque a veces:

- No necesitas una SPA
- No quieres recargar toda la página
- Quieres control sin sobreingeniería

F4 vive en ese punto medio.
